Module Guide

A GUI-friendly breakdown of the major layers, their responsibilities, and the most important implementation realities.

Layer 3: Orchestrator

Single entry point that classifies requests and coordinates research, memory, code context, drafting, verification, and output.

  • LangGraph workflow orchestration
  • Compliance and traceability
  • Spec-session and coding-task routing

Layer 2A: Loom

Shared automotive knowledge layer that centralizes curated ASAM/AUTOSAR facts, provenance, temporal state, retrieval, and migration management.

  • FalkorDB-backed property graph
  • Graphiti adapter boundary
  • Temporal state and evidence-chain APIs

Layer 2B: AMS

Local per-engineer memory system for continuity, steering, transcripts, and bounded resume context.

  • Retain / recall / reflect / resume
  • Per-objective continuity
  • Phase 2 integration target

Layer 2C: CMM

Repository-aware code structure module for call paths, impact analysis, architectural search, and code-grounded workflows.

  • search_graph / trace_call_path
  • Impact analysis and architecture lookup
  • Used when requests need repo reasoning
Current Strength The curated-source database foundation is live: structured rows, vectors, provenance chains, and temporal state bootstrap all work.
Current Fallback Community summaries are generated and searched locally from a cache file because direct `CommunitySummary` graph writes remain pathological.
Current Blocker Provider-backed Graphiti mutation and benchmark execution still require OpenAI or Azure OpenAI credentials in the environment.

Implementation Notes

This dashboard is meant to be friendly for architecture review, implementation planning, and stakeholder walkthroughs.

What is rendered here

The diagrams mirror the current `design.md` architecture plus the latest runtime realities: temporal fallback state, GraphRAG baseline retrieval, and community cache fallback.

Where the source lives

The main source of truth remains design.md. This HTML file is a browser-friendly presentation layer for that architecture.

How to use it

Use the sidebar to jump between views, the buttons to zoom or expand diagrams, and the theme toggle for dark/light review modes.

Saved alongside the spec documents so it can be versioned with the architecture itself.
